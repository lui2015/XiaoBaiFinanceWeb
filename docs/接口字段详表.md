# 小白财经 接口字段详表

> 版本：v1.0  
> 作用：定义 C 端公开接口、C 端用户接口、B 端管理接口的请求/响应/错误码与公共约定。

---

## 0. 公共约定

### 0.1 基础规范

- BaseURL（示例）
  - C 端公开：`https://api.xiaobaifinance.com`
  - C 端用户：`https://api.xiaobaifinance.com/u`
  - B 端管理：`https://admin-api.xiaobaifinance.com/admin`
- 请求/响应：`Content-Type: application/json; charset=utf-8`，时间字段统一 ISO8601（`2026-06-28T10:30:00+08:00`）。
- 全站强制 HTTPS，开启 HSTS。

### 0.2 公共请求头

| Header | 必填 | 说明 |
| --- | --- | --- |
| `Authorization` | C 端用户/B 端写接口必填 | `Bearer <accessToken>` |
| `X-Request-Id` | 否 | 客户端可生成 UUID，便于排障 |
| `X-Trace-Id` | 否 | 服务端透传 |
| `Idempotency-Key` | 写操作建议 | UUID，服务端在 5min 内幂等 |
| `X-CSRF-Token` | B 端写接口必填 | 与 Cookie 中 `csrfToken` 双重校验 |
| `Accept-Language` | 否 | 一期固定 zh-CN |
| `User-Agent` | 否 | 用于风控 |

### 0.3 公共响应结构

```json
{
  "code": 0,
  "message": "success",
  "data": { },
  "traceId": "8f0e..."
}
```

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| code | int | 0 表示成功；非 0 见错误码表 |
| message | string | 用户可读消息（错误时使用） |
| data | object/array/null | 业务数据 |
| traceId | string | 请求追踪 ID |

### 0.4 分页结构

请求参数：`page`（默认 1，从 1 开始）、`size`（默认 20，最大 50）、`sort`（如 `publishAt,desc`）。

响应：

```json
{
  "list": [],
  "page": 1,
  "size": 20,
  "total": 312,
  "totalPages": 16
}
```

### 0.5 错误码表

| code | HTTP | 含义 | 备注 |
| --- | --- | --- | --- |
| 0 | 200 | 成功 | |
| 1001 | 400 | 参数缺失或格式错误 | data 中返回字段名 |
| 1002 | 400 | 参数取值不合法 | |
| 1003 | 413 | 请求体/文件过大 | |
| 1010 | 429 | 触发限流 | 含 `retryAfter` 秒 |
| 2001 | 401 | 未登录或 Token 失效 | |
| 2002 | 401 | Token 过期，需 refresh | |
| 2003 | 403 | 无权限 / 越权 | |
| 2004 | 423 | 账号锁定 | 含 `unlockAt` |
| 2005 | 403 | 账号已被封禁 | 含 `reason / until` |
| 3001 | 404 | 资源不存在 | |
| 3002 | 409 | 资源冲突（如重复收藏、用户名重复） | |
| 3003 | 410 | 资源已下架/删除 | |
| 4001 | 400 | 短信验证码错误或过期 | |
| 4002 | 429 | 短信发送过频 | |
| 4003 | 400 | 图形验证码错误 | |
| 5001 | 422 | 文件类型/魔数不合法 | |
| 5002 | 422 | HTML/MD 内容净化失败 | |
| 9000 | 500 | 服务端内部错误 | |
| 9001 | 502/503 | 上游不可用 | |

### 0.6 限流策略（节选）

| 接口 | 维度 | 阈值 |
| --- | --- | --- |
| `POST /auth/sms/send` | 手机号 | 60s/次；单日 ≤ 10 |
| `POST /auth/sms/send` | IP | 60 次/小时 |
| `POST /auth/login/*` | 账号 | 5 次失败锁 10 min |
| 收藏/点赞/反馈 | 用户 | 10 次/分钟 |
| 通用读接口 | IP | 600 次/分钟 |
| 上传接口 | 管理员 | 20 次/分钟 |

## 2. C 端用户接口

> 写操作需 `Authorization` + `Idempotency-Key`；登录态使用 Access Token + Refresh Token Cookie 方案。

### 2.1 POST /auth/sms/send 发送短信验证码

**Body**

```json
{
  "phone": "13812345678",
  "scene": "login",
  "captcha": {
    "id": "geetest_xxx",
    "result": "{...}"
  }
}
```

| 字段 | 类型 | 必填 | 校验/说明 |
| --- | --- | --- | --- |
| phone | string | 是 | 中国大陆 11 位手机号 |
| scene | string | 是 | `login` / `bind` / `cancel` |
| captcha | object | 否 | 触发风控时必填（图形验证码） |

**Response.data**

```json
{ "ttl": 300, "cooldown": 60 }
```

**错误**：`4002` 限流；`4003` 图形验证码错误；`1002` 手机号格式错误。

---

### 2.2 POST /auth/login/sms 短信验证码登录/注册

**Body**

```json
{
  "phone": "13812345678",
  "code": "123456",
  "agreement": true
}
```

| 字段 | 类型 | 必填 | 校验 |
| --- | --- | --- | --- |
| phone | string | 是 | |
| code | string | 是 | 6 位数字 |
| agreement | bool | 是 | 必须为 true（已勾选协议） |

**Response.data**

```json
{
  "accessToken": "eyJhbGciOi...",
  "expiresIn": 900,
  "tokenType": "Bearer",
  "user": {
    "id": 1234,
    "nickname": "小白用户_8821",
    "avatarUrl": "https://cdn.xxx/avatar/default-12.png",
    "phone": "138****8821",
    "registeredAt": "2026-06-01T12:00:00+08:00",
    "isNewUser": false
  }
}
```

> Refresh Token 通过 `Set-Cookie: rt=...; HttpOnly; Secure; SameSite=Lax; Path=/auth; Max-Age=1209600` 下发。

---

### 2.3 POST /auth/login/password 邮箱密码登录

**Body**

```json
{ "email": "user@example.com", "password": "P@ssw0rd!", "agreement": true }
```

校验：邮箱 RFC 格式；密码 8–32 位、字母 + 数字。

返回结构同 2.2。

---

### 2.4 POST /auth/refresh 刷新 Token

- 不需要 Body，使用 Cookie 中 `rt`。
- 返回新的 `accessToken` + 新的 `rt` Cookie（轮换）。

**Response.data**

```json
{ "accessToken": "...", "expiresIn": 900 }
```

**错误**：`2002` Refresh Token 过期，需重新登录。

---

### 2.5 POST /auth/logout 退出登录

- 清除服务端 Refresh Token；返回空 data；前端清理本地态。

## 3. B 端管理接口

> 全部需 `Authorization`(B 端 Token) + `X-CSRF-Token`；管理后台与 C 端 Token 体系隔离。

### 3.1 POST /admin/login

**Body**

```json
{ "username": "admin", "password": "P@ss", "captcha": { "id":"...", "result":"..."} }
```

校验：失败 ≥3 次后必须带图形验证码；失败 5 次锁 10 min。

**Response.data**：`{ accessToken, expiresIn, admin: {id, username, role} }`，下发 Refresh Cookie。

---

### 3.2 文章管理

#### 3.2.1 POST /admin/articles 创建文章

`Content-Type: multipart/form-data`

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| file | binary | source=html/markdown 必填 | `.html ≤2MB` / `.md ≤1MB` |
| meta | json string | 是 | 见下 |

`meta`：

```json
{
  "title": "一文读懂市盈率(PE)",
  "slug": "what-is-pe",
  "summary": "...",
  "categoryId": 21,
  "subCategoryId": 211,
  "tags": ["估值", "PE"],
  "coverUrl": "https://cdn.../p/101.webp",
  "sourceType": 1,
  "status": 0,
  "publishAt": "2026-06-28T10:00:00+08:00",
  "contentMd": "# ...",
  "contentHtml": "<h2>...</h2>"
}
```

| 字段 | 校验 |
| --- | --- |
| title | 1–60 字 |
| slug | 仅 a-z0-9-，2–80 字，唯一；为空时服务端按 title 自动生成 |
| summary | ≤120 字；为空自动截取正文 120 字 |
| categoryId | 必填 |
| tags | 数组，最多 5；自动按 name 创建/复用 |
| sourceType | 0 HTML / 1 Markdown / 2 在线编辑 |
| status | 0 草稿 / 1 已发布 / 2 已下架 |
| publishAt | 仅 status=1 时生效 |
| contentMd | sourceType=1 必填，≤500KB |
| contentHtml | sourceType=0/2 必填 |

**安全处理**

- 服务端：扩展名 + MIME + 魔数三重校验；HTML 走白名单净化；Markdown 渲染后再走净化。
- 上传图片自动抽取并存对象存储，替换为站内 URL。

**Response.data**

```json
{
  "id": 101,
  "slug": "what-is-pe",
  "status": 0,
  "sanitizeReport": {
    "removedTags": [{"tag": "script", "count": 2}],
    "removedAttrs": [{"attr": "onerror", "count": 1}],
    "rewroteImages": 3
  }
}
```

**错误**：`5001` 文件不合法；`5002` 净化失败；`3002` slug 重复。

#### 3.2.2 PUT /admin/articles/{id} 编辑

Body 同上 `meta`，可省略未变字段；`file` 可选。

#### 3.2.3 PATCH /admin/articles/{id}/status

**Body**：`{ "status": 1 }`（0 草稿 / 1 已发布 / 2 已下架）

#### 3.2.4 PATCH /admin/articles/{id}/pin

置顶：`{ "pinned": true, "pinSort": 1 }`（同分类内）。

#### 3.2.5 DELETE /admin/articles/{id} 软删除

支持 `?hard=true`（仅超管，30 天后允许）。

#### 3.2.6 GET /admin/articles 列表查询

**Query**

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| keyword | string | 标题模糊 |
| categoryId | long | |
| status | int | 多选用逗号分隔 |
| from / to | datetime | 发布时间范围 |
| page / size / sort | | |

**Response.data**：分页，每项含 `id, title, slug, category, status, viewCount, favoriteCount, publishAt, updatedAt, sourceType`。

#### 3.2.7 POST /admin/articles/upload-image 富文本/编辑器贴图

`multipart`：`file` 字段，返回 `{ url, width, height, size }`。

## 5. 安全要求小结（强制）

1. 所有 SQL 使用参数化（Prepared Statement / ORM 占位符），严禁字符串拼接。
2. 所有写接口校验登录态 + 角色 + 资源所属（如收藏/反馈仅本人可读写）。
3. HTML / Markdown 渲染产物均经过白名单净化：禁止 `<script>` / `<iframe>` / `on*` / `javascript:` 等。
4. 文件上传校验：扩展名 + MIME + 魔数；图片转存站内 CDN；最大体积如 §3.2.1。
5. SSRF：服务端抓取外链时拒绝 `127/8、10/8、172.16/12、192.168/16、169.254/16` 等内网网段。
6. 敏感信息（手机号/邮箱）落库前 AES-GCM 加密 + 哈希；日志严禁打印 Token / 密码 / 验证码。
7. JWT：HS256 或 RS256；密钥仅环境变量；签发含 `iat/exp/jti`；登出/注销使 jti 黑名单。
8. CSRF：B 端使用 Cookie + 自定义 Header 双校验；C 端 SameSite=Lax + 短期 Token。
9. 限流：见 §0.6；命中后返回 429 + `Retry-After`。
10. 反爬虫：搜索、详情等高频接口对异常 UA / 高频 IP 做风控验证码或封禁。
