# 对外投稿上传接口（开放平台）

面向 AI / 第三方系统的内容投稿接口。需使用「设置 → 开放平台」中生成的 **API Key** 调用，提交的内容会经过安全净化后**直接发布**，即刻在「小白财经」平台公开展示，无需人工审核。每次成功调用都会计入该用户的「当日 / 累计」调用次数。

> 📄 本文档在线地址（公开只读，可直接分享或交给 AI 阅读并生成调用代码）：
> `GET https://www.luliming.xyz/XiaoBaiFinance/api/open/docs`
>
> ⚠️ 所有接口路径均带有站点前缀 `/XiaoBaiFinance`（basePath），调用时不要漏掉。

## 1. 接口说明

```
POST https://www.luliming.xyz/XiaoBaiFinance/api/open/articles
Content-Type: application/json
Authorization: Bearer <你的开放平台 API Key>
```

> 需要鉴权：在请求头携带 `Authorization: Bearer <key>`（或 `x-api-key: <key>`）。
> API Key 在「设置 → 开放平台」页面生成；缺少或无效的 Key 会返回 `401`。

### 获取 API Key

1. 登录后进入「设置 → 开放平台」。
2. 点击「创建新密钥」，保存返回的明文密钥（仅展示一次）。
3. 把页面上的「AI 提示词」复制给你的 AI，AI 即可按规范调用本接口投稿。

### 请求体字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `title` | string(2–60) | 是 | 标题 |
| `contentHtml` | string(≤200000) | 是 | HTML 正文（`sourceType=0` 时使用，默认即为此模式） |
| `categorySlug` | string | 与 `categoryId` 二选一 | 一级分类 slug |
| `categoryId` | string(数字) | 与 `categorySlug` 二选一 | 一级分类 ID |
| `subCategoryId` | string(数字) | 否 | 二级分类 ID |
| `summary` | string(≤120) | 否 | 摘要，缺省自动截取正文 |
| `coverUrl` | string(url) | 否 | 封面图 URL |
| `slug` | string | 否 | 自定义 slug，冲突时自动追加随机后缀 |
| `tags` | string[] | 否 | 标签名称，最多 5 个，服务端自动创建 |
| `sourceType` | 0 \| 1 | 否 | 0=HTML（默认），1=Markdown；填 1 时改用 `contentMd` 字段 |
| `contentMd` | string(≤200000) | `sourceType=1` 时必填 | Markdown 正文 |

> ⚠️ `categorySlug` 必须填**下表中真实存在且启用的 slug**，否则会返回 `400 分类不存在或未启用`。这是投稿失败最常见的原因。

### 可用分类（线上现网）

一级分类（`categorySlug` 取下列 slug）：

| id | 名称 | slug |
|---|---|---|
| 1 | 基础概念 | `basics` |
| 4 | 基本面分析 | `fundamental` |
| 7 | 技术面分析 | `technical` |
| 10 | 宏观经济 | `macro` |
| 11 | 投资品种 | `instruments` |
| 15 | 理财规划 | `planning` |
| 16 | 行为金融 | `behavioral` |

二级分类通过 `subCategoryId`（数字 ID）指定，例如：`2` 名词解释、`3` 入门指南（属基础概念），`5` 财报解读、`6` 行业分析（属基本面分析），`8` K线形态、`9` 指标（属技术面分析），`12` 股票、`13` 基金、`14` 债券（属投资品种）。

### 成功响应

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "id": "123",
    "slug": "my-post",
    "status": 1,
    "message": "投稿已发布"
  },
  "traceId": "..."
}
```

### 常见错误码

| HTTP | 典型 message | 含义与排查 |
|---|---|---|
| 401 | `缺少 API Key，请在请求头携带 ...` / `API Key 无效或已停用` | 未携带密钥、密钥错误，或密钥已被吊销/停用 |
| 400 | `缺少必填字段：title` / `contentHtml 必填` | 缺少必填字段或字段名拼写错误 |
| 400 | `分类不存在或未启用` | `categorySlug`/`categoryId` 不在「可用分类」表中，**最常见错误**，请照上表填写 |
| 400 | `需要提供 categoryId 或 categorySlug` | 两者都没传，至少提供其一 |
| 400 | `正文内容为空或被安全过滤` | `contentHtml` 净化后为空（例如整段都是被过滤的标签） |
| 429 | `提交过于频繁，请稍后再试` | 同一来源 IP 每分钟超过 20 次，或同一密钥每分钟超过 60 次，稍后重试 |

> 只有返回 `"code": 0` 才代表投稿成功、文章已发布并可在后台文章列表看到；其余任何返回都表示**未创建文章**。

## 2. 安全策略

- **API Key 鉴权**：必须携带有效的开放平台密钥（`Authorization: Bearer <key>`），密钥以 sha256 哈希存储，明文仅创建时展示一次。
- **内容净化**：HTML/Markdown 均经 `sanitize` 白名单过滤，剔除 `<script>`、`on*` 事件、`javascript:` 等 XSS 向量。
- **直接发布**：所有投稿 `status=1` 落库并立即公开展示（无需人工审核）。
- **限流**：同一来源 IP 每分钟最多 20 次，同一密钥每分钟最多 60 次。
- **调用统计**：每次成功调用都会累加该用户的「当日 / 累计」调用次数，可在「开放平台」页面查看。
- **审计**：每次投稿写入 `operation_log`（`action=open.article.submit`，记录密钥 / 用户 / IP / UA）。

## 3. 调用示例

```bash
curl -X POST "https://www.luliming.xyz/XiaoBaiFinance/api/open/articles" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer xbk_live_你的密钥" \
  -d '{
    "title": "一篇投稿文章",
    "contentHtml": "<h2>小标题</h2><p>正文内容……</p>",
    "categorySlug": "instruments",
    "subCategoryId": "12",
    "summary": "这是文章摘要",
    "coverUrl": "https://cdn.example.com/cover.webp",
    "tags": ["投资", "理财"]
  }'
```
