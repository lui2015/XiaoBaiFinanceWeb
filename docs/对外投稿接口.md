# 对外投稿上传接口

面向第三方 / 外部系统的内容投稿接口。调用方使用 **API Key** 鉴权，提交的内容会经过安全净化后落库为**草稿（待审核）**，由管理员在后台审核后再发布。

> 📄 本文档在线地址（公开只读，可直接分享或交给 AI 阅读并生成调用代码）：
> `GET https://your-domain/XiaoBaiFinance/api/open/docs`
>
> ⚠️ 所有接口路径均带有站点前缀 `/XiaoBaiFinance`（basePath），调用时不要漏掉。

## 1. 启用与配置

Key 只从环境变量读取，**不落库、不进代码**。在部署环境（如 `.env` / 容器环境变量）中配置：

```bash
# 多把 key 用英文逗号分隔；每把 key 建议 ≥ 32 位随机串
OPEN_API_KEYS=xbk_live_9f3c...a1,xbk_live_7d21...b8
```

生成一把随机 key 的示例：

```bash
node -e "console.log('xbk_live_' + require('crypto').randomBytes(24).toString('hex'))"
```

> 未配置 `OPEN_API_KEYS` 时，接口默认关闭并返回 403，避免误开放。

## 2. 接口说明

```
POST https://your-domain/XiaoBaiFinance/api/open/articles
Content-Type: application/json
```

鉴权头（二选一）：

```
Authorization: Bearer <API_KEY>
# 或
X-API-Key: <API_KEY>
```

### 请求体字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `title` | string(2–60) | 是 | 标题 |
| `sourceType` | 0 \| 1 | 否 | 0=HTML，1=Markdown，默认 1 |
| `contentMd` | string | `sourceType=1` 时必填 | Markdown 正文（≤ 200000 字符） |
| `contentHtml` | string | `sourceType=0` 时必填 | HTML 正文（≤ 200000 字符） |
| `categoryId` | string(数字) | 与 `categorySlug` 二选一 | 分类 ID |
| `categorySlug` | string | 与 `categoryId` 二选一 | 分类 slug |
| `subCategoryId` | string(数字) | 否 | 子分类 ID |
| `summary` | string(≤120) | 否 | 摘要，缺省自动截取正文 |
| `slug` | string | 否 | 自定义 slug，冲突时自动追加随机后缀 |
| `coverUrl` | string(url) | 否 | 封面图 URL |
| `tags` | string[] | 否 | 标签名称，最多 5 个，服务端自动创建 |

### 成功响应

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "id": "123",
    "slug": "my-post",
    "status": 0,
    "message": "投稿已提交，等待管理员审核后发布"
  },
  "traceId": "..."
}
```

### 常见错误码

| HTTP | code | 含义 |
|---|---|---|
| 401 | 1002 | 缺少或无效的 API Key |
| 403 | 1003 | 对外接口未启用 |
| 400 | 1001 / 2001 | 参数错误 / 校验失败 |
| 429 | 1007 | 触发限流 |

## 3. 安全策略

- **内容净化**：Markdown/HTML 均经 `sanitize` 白名单过滤，剔除 `<script>`、`on*` 事件、`javascript:` 等 XSS 向量。
- **强制草稿**：所有投稿 `status=0`，不可通过本接口直接发布，需管理员审核。
- **限流**：单 Key 每分钟 30 次、单 IP 每分钟 60 次。
- **审计**：每次投稿写入 `operation_log`（`action=open.article.submit`，记录 keyRef/IP/UA），keyRef 为 key 的 SHA-256 前缀，不含明文。

## 4. 调用示例

```bash
curl -X POST "https://your-domain/XiaoBaiFinance/api/open/articles" \
  -H "Authorization: Bearer $OPEN_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "一篇投稿文章",
    "sourceType": 1,
    "contentMd": "# 标题\n\n正文内容……",
    "categorySlug": "market",
    "tags": ["投资", "理财"]
  }'
```
