# 对外投稿上传接口

面向第三方 / 外部系统的内容投稿接口。**无需鉴权、任何人可直接调用**，提交的内容会经过安全净化后落库为**草稿（待审核）**，由管理员在后台审核后再发布。

> 📄 本文档在线地址（公开只读，可直接分享或交给 AI 阅读并生成调用代码）：
> `GET https://www.luliming.xyz/XiaoBaiFinance/api/open/docs`
>
> ⚠️ 所有接口路径均带有站点前缀 `/XiaoBaiFinance`（basePath），调用时不要漏掉。

## 1. 接口说明

```
POST https://www.luliming.xyz/XiaoBaiFinance/api/open/articles
Content-Type: application/json
```

> 无需任何鉴权头，直接 POST JSON 即可。

### 请求体字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `title` | string(2–60) | 是 | 标题 |
| `contentHtml` | string(≤200000) | 是 | HTML 正文（`sourceType=0` 时使用，默认即为此模式） |
| `categorySlug` | string | 与 `categoryId` 二选一 | 一级分类 slug |
| `categoryId` | string(数字) | 与 `categorySlug` 二选一 | 一级分类 ID |
| `subCategoryId` | string(数字) | 否 | 二级分类 ID |
| `summary` | string(≤120) | 否 | 摘要，缺省自动截取正文 |
| `coverUrl` | string(url) | 否 | 封面图 URL |
| `slug` | string | 否 | 自定义 slug，冲突时自动追加随机后缀 |
| `tags` | string[] | 否 | 标签名称，最多 5 个，服务端自动创建 |
| `sourceType` | 0 \| 1 | 否 | 0=HTML（默认），1=Markdown；填 1 时改用 `contentMd` 字段 |
| `contentMd` | string(≤200000) | `sourceType=1` 时必填 | Markdown 正文 |

### 成功响应

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "id": "123",
    "slug": "my-post",
    "status": 0,
    "message": "投稿已提交，等待管理员审核后发布"
  },
  "traceId": "..."
}
```

### 常见错误码

| HTTP | code | 含义 |
|---|---|---|
| 400 | 1001 / 2001 | 参数错误 / 校验失败（如缺少标题、正文、分类等） |
| 429 | 1007 | 触发限流（提交过于频繁） |

## 2. 安全策略

- **无鉴权公开接口**：任何人均可调用，因此以下几道防护尤为关键。
- **内容净化**：HTML/Markdown 均经 `sanitize` 白名单过滤，剔除 `<script>`、`on*` 事件、`javascript:` 等 XSS 向量。
- **强制草稿**：所有投稿 `status=0`，不可通过本接口直接发布，**必须由管理员在后台审核后才会公开**。
- **限流**：同一来源 IP 每分钟最多 20 次提交。
- **审计**：每次投稿写入 `operation_log`（`action=open.article.submit`，记录来源标记 / IP / UA）。

## 3. 调用示例

```bash
curl -X POST "https://www.luliming.xyz/XiaoBaiFinance/api/open/articles" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "一篇投稿文章",
    "contentHtml": "<h2>小标题</h2><p>正文内容……</p>",
    "categorySlug": "market",
    "subCategoryId": "21",
    "summary": "这是文章摘要",
    "coverUrl": "https://cdn.example.com/cover.webp",
    "tags": ["投资", "理财"]
  }'
```
